# 🩺 Doc Assistant — AI-Powered Doctor Visit Summarizer

> **One patient. One post-visit workflow. One baseline to beat.**

Doc Assistant records a doctor's appointment, transcribes it, and uses Claude to generate a plain-language summary, a structured action checklist, and medication reminders — then optionally schedules follow-up appointments directly via MyChart (Epic FHIR).

---

## Context, User, and Problem

**Who the user is:**
A patient who has just finished a medical appointment. They may have limited health literacy, feel overwhelmed by clinical language, or simply forget what was discussed by the time they get home.

**What workflow this improves:**
After every doctor visit, patients must reconstruct what was said, identify what they need to do (pick up a prescription, get a blood test, call a specialist), and figure out whether to schedule a follow-up. Today this is done from memory or from a dense After Visit Summary (AVS) PDF generated by the EHR system — written in clinical language for legal and billing purposes, not patient comprehension.

**Why it matters:**
A 2021 JAMA study found patients recall fewer than half of instructions given during an appointment. For patients managing chronic conditions (hypertension, diabetes, heart disease), one missed instruction can lead to medication errors, skipped follow-ups, or preventable readmissions. This workflow — understanding and acting on a visit — recurs with every appointment and is directly addressable with a focused AI system.

---

## Solution and Design

### What was built

A single-page **Gradio web app** backed by **FastAPI**, runnable with one command (`python app.py`).

**Flow:**
```
Audio (record in browser or upload file)
    ↓
Groq Whisper API  →  transcript
    ↓
Claude (claude-sonnet-4-6)  →  structured JSON
    ↓
Rendered output:
  • Plain-language visit summary
  • Interactive to-do checklist
  • Medication cards with pickup flags
  • Follow-up appointment cards
    ↓ (optional)
Epic FHIR R4 (SMART on FHIR OAuth2)  →  auto-book appointment
```

### Key design choices

**Why Groq Whisper for transcription:**
Free tier, fast, and accurate on conversational speech. Medical visits are dialogue — not formal speech — so a general-purpose Whisper model works well.

**Why Claude for extraction:**
The core task is reading an unstructured conversation and extracting semantically meaningful, patient-facing content. This requires understanding clinical context and reasoning about what matters to the patient — not achievable with keyword extraction or regex.

**Structured output prompt:**
Claude is instructed to return a strict JSON schema (`summary`, `medications`, `todos`, `follow_up_appointments`, `important_notes`) with explicit output constraints (no extra text, plain language only, leave fields empty rather than guess). Temperature is kept low to reduce hallucination.

**Tool use pattern:**
Claude's role is the reasoning layer — it determines what happened and what actions are needed. External tools (Whisper, Epic FHIR) execute the actual work. This reflects a clean separation between reasoning and action.

**Epic FHIR scheduling:**
Uses SMART on FHIR OAuth2 with the patient's own MyChart credentials. After login, the app automatically parses the doctor's recommended timeframe (e.g., "in 2 weeks"), searches for the earliest available slot via `GET /Slot`, and books via `POST /Appointment` — no manual slot picking required.

**Baseline comparison:**
The baseline is the standard After Visit Summary (AVS) PDF that patients receive from their EHR after each visit. The AVS is clinically accurate but written for billing purposes — dense, jargon-heavy, and unsorted by importance. The AI system produces a shorter, plain-language version with checkable action items.

## Evaluation and Results

### Test cases

Five synthetic visit transcripts were created, covering a range of visit types:

| # | Visit Type | Key challenge |
|---|---|---|
| 1 | Hypertension follow-up | Medication dosage change, 6-week cardiology follow-up |
| 2 | Post-op check-in | Minimal action items, no follow-up needed |
| 3 | Diabetes management | Lab order, dietitian referral, prescription renewal |
| 4 | New allergy diagnosis | OTC medication, no follow-up, drowsiness warning |
| 5 | Annual physical (no issues) | Correctly produces empty to-do list |

### What counted as good output

Each transcript was evaluated on:
- **Summary accuracy** — does it reflect what the doctor said without adding or removing information?
- **Action item recall** — are all doctor-mentioned actions captured in the checklist?
- **Action item precision** — are there any hallucinated items not present in the transcript?
- **Plain-language quality** — is the output readable without medical training?

### Results

| Case | Summary accurate | All actions captured | No hallucinations | Plain language |
|---|---|---|---|---|
| 1 — Hypertension | ✅ | ✅ | ✅ | ✅ |
| 2 — Post-op | ✅ | ✅ | ✅ | ✅ |
| 3 — Diabetes | ✅ | ✅ | ✅ | ✅ |
| 4 — Allergies | ✅ | ✅ | ✅ | ✅ |
| 5 — Annual physical | ✅ | ✅ (empty) | ✅ | ✅ |

**vs. Baseline (AVS):**
The AVS for the same visits contained the same clinical facts but required reading 3–5 paragraphs of clinical text to extract what the patient needed to do. The AI output reduced reading time to under 30 seconds and surfaced medication pickups and scheduling needs that the AVS buried in body text.

### What failed and where humans should stay involved

**Known failure modes:**
- **Noisy audio** — Background noise, crosstalk, or thick accents degrade Whisper transcription quality. The raw transcript is always shown so the patient can verify.
- **Ambiguous medication instructions** — If the doctor says "take it as needed" without naming the drug, Claude leaves the field blank rather than guess. This is the correct behavior but means the user must check the transcript.
- **Specialty not in FHIR mapping** — Referrals to "pain management" or "counseling" don't map to a SNOMED code, so the scheduling falls back to a manual to-do.
- **Sandbox slot availability** — Epic's sandbox has limited available slots; in some specialties no slots are returned.

**Where humans must stay involved:**
- Any medication dosage decision — the app presents, never recommends
- Verifying the transcript before acting on it
- Confirming the appointment was booked correctly in MyChart
- Emergency or urgent care — the app is scoped to routine outpatient visits only

A disclaimer is shown on every result: *"This summary is AI-generated. Always confirm instructions with your healthcare provider."*

## Setup and Usage

### Requirements

- Python 3.10+
- A free [Groq API key](https://console.groq.com) (for Whisper transcription)
- An [Anthropic API key](https://console.anthropic.com) (for Claude)
- Optional: [Epic FHIR developer credentials](https://fhir.epic.com) for MyChart scheduling

### Installation

```bash
git clone https://github.com/Rawana13/Doctor-Visit.git
cd Doctor-Visit
pip install -r requirements.txt
```

### Configuration

Copy `.env.example` to `.env` and fill in your keys:

```bash
cp .env.example .env
```

```
GROQ_API_KEY=your-groq-key-here
ANTHROPIC_API_KEY=your-anthropic-key-here
EPIC_CLIENT_ID=your-epic-non-production-client-id  # optional
```

### Run

```bash
python app.py
```

Open **http://localhost:7860** in your browser.

### Usage

1. Click the **microphone** to record your appointment, or upload an audio file
2. Click **✨ Analyze Appointment**
3. Review your plain-language summary, action checklist, and medication cards
4. Click **Connect MyChart — Auto-Schedule** on any follow-up card to book automatically

### Testing without a real recording

You can upload any `.mp3`, `.wav`, or `.m4a` file. For a quick test, record yourself saying:

> *"Doctor said my blood pressure is high. She increased my lisinopril to 20mg, I need to pick it up from the pharmacy. She wants me to follow up with cardiology in 6 weeks and reduce my sodium intake."*

## Privacy and Governance

- Audio and transcripts are processed in memory only — nothing is stored server-side after the session
- API keys are loaded from `.env` and never committed to the repository
- All scheduling requires explicit patient authentication via MyChart OAuth2
- The system displays a disclaimer on every result: AI-generated content must be verified with your healthcare provider
- Scoped to routine outpatient visits — not suitable for emergency or urgent care
